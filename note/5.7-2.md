关键点是：

```text

```

如果你的 text 是用 `/home/hbxz_lzl/clip-vit-large-patch14` 提的，那 RGB 也用这个目录提，这样 image encoder 和 text encoder 的 projection space 才天然一致。不要混成：

那样就不是同一个 CLIP 空间了。

# 二、方向 A：CLIP-L RGB / Frame Image Feature

## 1. 核心想法

原来 LSTR 的 RGB 特征是动作识别预训练特征，偏动作判别；text feature 是 CLIP-L 提的，偏语义空间。
如果 RGB/frame image 也用同一个 CLIP-L 提，那么：

```text
CLIP-L image feature ↔ CLIP-L class text feature
```

会天然在同一语义空间，更适合做 video-text similarity、top-k candidate selection、text residual 或 semantic fusion。

不过它也有风险：

```text
CLIP 单帧图像特征更强在静态语义，不一定比原始 Kinetics RGB/Flow 更懂动作时序。
```

所以第一步不要直接替换原 RGB，而是先做诊断。

## 2. 第一阶段：只做诊断，不改 LSTR

任务：

```text
用 /home/hbxz_lzl/clip-vit-large-patch14 提取 THUMOS 测试集/训练集 frame image feature。
同时确认 class text feature 也来自同一个 CLIP-L/14。
```

然后统计：

```text
1. action-only frame 的 CLIP image-text top-1 hit
2. action-only frame 的 top-3 hit
3. action-only frame 的 top-5 hit
4. GT class 在 CLIP image-text similarity 里的 rank 分布
5. 只用 CLIP image-text similarity 的 zero-shot per-frame AP / mAP
```

目的：

```text
先判断 CLIP-L image feature 对 THUMOS 动作类别有没有判别价值。
```

如果诊断很差，就不要把它加进 LSTR；
如果诊断不错，再做第二阶段。

## 3. 第二阶段：作为第四模态，不直接替换 RGB

更稳的设计是：

```text
原始三模态：
RGB + Flow + evidence text

新四模态：
RGB + Flow + evidence text + CLIP-L image feature
```

不要一开始替换 RGB，因为原 RGB/Flow 仍然是动作判别主力。CLIP-L image feature 先作为语义补充。

第一版建议：

```text
冻结原始 LSTR 主路；
新增 CLIP-image adapter；
只训练 adapter 或 adapter + classifier；
观察 raw mAP 是否至少不低于 71.518。
```

## 4. 第三阶段：CLIP-L image 用于 text residual

如果 CLIP-L image-text top-k 命中率好，可以把当前 Exp4-v8 的 residual 从：

```text
LSTR hidden ↔ class text
```

改成：

```text
CLIP-L image feature ↔ class text
```

也就是：

```text
final_logits = base_logits + beta * topk_masked_sim(CLIP_image, CLIP_text)
```

这可能比用 LSTR hidden 做 similarity 更合理，因为二者来自同一个 CLIP 空间。

# 三、方向 B：Self / Teacher Distillation

## 1. “自己蒸馏自己”可行吗？

可行，但不能是简单地：

```text
同一个输出蒸馏同一个输出
```

那没有意义。

必须满足一个条件：

```text
teacher 分支要比 student 更稳定，或者 teacher 和 student 是不同视角/不同粒度。
```

FineCLIP 里的 real-time self-distillation 就不是简单复制自己，而是用当前模型的全局 region crop embedding 去指导 pooled region dense feature；同时它还保留 global contrastive learning 来维持 visual-semantic consistency。论文明确强调 FineCLIP 通过 global contrastive、regional contrastive 和 real-time self-distillation 三个目标共同工作，其中 self-distillation 是把 global representation capability 传给 local dense features。

FineCLIP 还指出，单纯 frozen teacher 会限制 student 上限，所以它用 real-time self-distillation，让 trainable model 在训练中“自己教自己”。但这依赖它有不同粒度的表征：global crop embedding 和 pooled local feature，不是完全同一个 logits。

迁移到我们这里，可以有三种蒸馏版本。

---

## 2. 版本 KD-A：Frozen teacher，最稳

Teacher：

```text
原始三模态 epoch-14 checkpoint
raw = 71.518
TTA = 71.950
冻结，不更新
```

Student：

```text
加入 text residual / CLIP-L image branch / fusion module 的模型
```

损失：

```text
L = L_cls + λ_kd * KL(student_logits / T, teacher_logits / T)
```

推荐参数：

```text
T = 2.0
λ_kd = 0.05 / 0.10 / 0.30
```

作用：

```text
让新模块学习时不要偏离原始强 baseline 太远。
```

这是目前最适合你的版本，因为我们之前很多新模块的问题就是容易扰动原主路。

---

## 3. 版本 KD-B：EMA teacher，更像 ALBEF

Teacher 不是固定的原模型，而是 student 的 EMA 版本：

```text
teacher_params = momentum * teacher_params + (1 - momentum) * student_params
```

这更接近 ALBEF 的 momentum distillation 思路。ALBEF 的核心就是 align before fuse，并使用 momentum distillation 产生更稳定的 soft targets。这个想法可以参考，但代码复杂度比 frozen teacher 高。

建议：

```text
先不做 EMA teacher。
等 frozen teacher KD 有效果，再考虑 EMA。
```

---

## 4. 版本 KD-C：FineCLIP-style self-distill

这个更像 FineCLIP，不是 logits KD，而是不同粒度 feature distillation：

```text
全局/窗口级强语义 feature 作为 teacher；
局部/当前帧 feature 作为 student；
用 cosine loss 或 KL 让局部帧特征继承全局语义。
```

对应 OAD 可以写成：

```text
window-level CLIP/LSTR semantic feature → current-frame feature
```

但这个需要重新设计 feature pooling 和时间窗口，不适合马上做第一版。

---

# 四、如何避免“蒸馏涨了，贡献都是蒸馏”的问题

你这个担心非常重要。
所以蒸馏实验必须成组做，不能只跑一个 text+KD。

必须有这 4 个对照：

```text
A. Baseline
原始三模态，不加新模块，不加 KD

B. Distill only
只加 KD，不加 text residual / CLIP image branch

C. Text module only
只加你的 text residual / CLIP-L image branch，不加 KD

D. Text module + KD
你的模块 + KD
```

归因逻辑：

```text
如果 B 涨了，D 没比 B 高：
主要是 KD 的作用，不能说你的模块有效。

如果 C 有一点提升，B 有一点提升，D 最好：
说明 text module 和 KD 互补，你的模块有独立贡献。

如果 B 不涨，D 涨：
说明 KD 本身不是主因，新模块才是主因。
```

所以蒸馏可以用，但必须做好 `distill-only` 对照。
