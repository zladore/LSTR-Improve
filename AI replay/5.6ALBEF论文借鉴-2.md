这次实验不是照搬 ALBEF，而是抽象它的思想：

# 实验 1：Video-Text Contrastive Alignment

## 1. 核心想法

在原始 LSTR 训练基础上，加一个轻量 video-text 对齐辅助损失。

原本 LSTR hidden / visual feature 主要服务于分类，现在额外让它和动作类别文本特征对齐。

直观目标：

```text
当前帧/窗口的视频特征
应该更接近 GT 动作类别文本，
远离其他动作类别文本。
```

## 2. 形式

已有：

```text
H: LSTR hidden feature / classifier 前特征，形状约为 [B, T, D]
E: class text feature，形状约为 [C, Dt]
```

加入两个投影头：

```text
z_v = Proj_v(H)
z_t = Proj_t(E)
```

然后做相似度：

```text
sim[t, c] = cosine(z_v[t], z_t[c])
```

训练损失：

```text
L = L_cls + λ_align * L_align
```

如果 THUMOS 当前标签是单标签帧，可先用 CE：

```text
L_align = CE(sim[t, :], y_t)
```

如果实际是多标签格式，则改成 BCE 或 multi-positive contrastive。

## 3. 第一轮推荐参数

```text
λ_align = 0.01
λ_align = 0.03
λ_align = 0.05
```

不要一开始设太大，避免文本监督破坏原本 71.950 的视觉分类能力。

## 4. 实验目的

这组实验主要证明：

```text
文本特征可以约束视频特征空间；
不是只在测试时做 logits calibration 才有用。
```

## 5. 需要记录的指标

除了官方 mAP，还要记录：

```text
Action detection per-frame mAP
GT 类别在 video-text similarity 里的 rank
action-only frames 的 top-1 hit rate
action-only frames 的 top-3 hit rate
action-only frames 的 top-5 hit rate
```

# 实验 2：Text-Guided Gated Fusion

## 1. 核心想法

这是更像“特征融合”的实验。

不是只把 text logits 加到分类结果上，而是在 classifier 前，让视觉时序特征主动和文本特征交互。

但是不能粗暴融合，所以用 gated residual 保护主干。

## 2. 形式

视觉特征：

```text
F_v: [B, T, D]
```

文本特征：

```text
E: [C, Dt]
```

先投影到同一维度：

```text
F_v_proj = Proj_v(F_v)
E_proj = Proj_t(E)
```

做 cross-attention：

```text
C_t = CrossAttn(Q=F_v_proj, K=E_proj, V=E_proj)
```

再做门控：

```text
g = sigmoid(MLP([F_v_proj, C_t, F_v_proj * C_t]))
```

融合：

```text
F_fuse = LayerNorm(F_v_proj + λ_fuse * g * C_t)
```

最后 classifier 使用 `F_fuse`。

## 3. 关键保护策略

必须保留开关：

```text
MODEL.TEXT_FUSION.ENABLED False / True
```

关闭时必须完全等价于原始 71.950 主线。

推荐第一版设置：

```text
cross-attention 层数 = 1
gate 初始化接近 0
λ_fuse = 0.05 或 0.1
dropout = 0.1
```

gate 初始化建议偏保守，例如让 sigmoid 后初始值很小：

```text
gate bias = -4.0
```

这样初始时文本几乎不影响视觉主路，训练中如果文本有用，gate 再慢慢打开。

## 4. 实验目的

这组实验主要证明：

```text
文本不是简单拼接进去，而是可以作为语义查询/语义上下文，
对视频特征进行可控增强。
```

这比单纯 concat 更有“特征融合”的说服力。

## 5. 需要重点观察

```text
mAP 是否超过 71.950
gate mean 是否一直接近 0
gate 是否过大导致性能下降
text fusion 前后 logits 是否明显变化
相似动作类别混淆是否下降
```

如果 mAP 下降但 gate 很大，说明文本干扰太强；
如果 mAP 不变且 gate 接近 0，说明模型没有使用文本；
如果 mAP 小幅提升且 gate 保持中小值，说明融合可能有效。

# 实验 4：Top-k Text Candidate Fusion

## 1. 核心想法

这是实验 2 的更安全版本。

不是让所有类别文本都参与 cross-attention，而是先筛出 top-k 候选动作文本，只让这些文本进入融合。

这个思想来自：

```text
先粗筛候选语义，再做细粒度 video-text interaction。
```

## 2. 和之前 TTA_TOPK 的区别

之前的 top-k：

```text
logits-level / TTA-level top-k
主要用于推理阶段校准分类器输出。
```

这次的 top-k：

```text
feature-level top-k
在 classifier 前选择文本候选，让文本参与特征融合。
```

所以它不是简单复用原来的 TTA_TOPK，而是把 top-k 思想前移到特征交互阶段。

## 3. top-k 来源

第一版可以有两个选择。

### 方案 A：用当前视觉 logits 选 top-k

```text
topk_classes = TopK(cls_logits)
```

优点：

```text
实现简单，和当前分类器预测一致。
```

缺点：

```text
如果当前 logits 错了，top-k 文本也会错。
```

### 方案 B：用 video-text similarity 选 top-k

```text
topk_classes = TopK(cosine(z_v, z_t))
```

优点：

```text
更符合 video-text 对齐思想。
```

缺点：

```text
需要先保证 video-text similarity 有一定质量。
```

第一版建议：

```text
先做方案 A，代码风险最低。
如果实验 1 的对齐效果不错，再做方案 B。
```

## 4. 推荐参数

```text
topk = 3
topk = 5
```

先不要做太多：

```text
topk = 2 / 3 / 5
```

最多三组就够了，避免实验太散。

## 5. 融合形式

只把 top-k 文本送入 cross-attention：

```text
E_topk: [B, T, K, D]
C_t = CrossAttn(Q=F_v, K=E_topk, V=E_topk)
F_fuse = LayerNorm(F_v + λ_fuse * g * C_t)
```

注意 OAD 原则：

```text
每个时间步只能使用当前及过去视觉信息。
文本类别库是固定先验，可以使用；
但 top-k 选择不能依赖未来帧预测。
```

## 6. 实验目的

这组实验主要证明：

```text
相比所有类别文本无差别参与，
先筛候选动作文本再融合更稳定、更符合 OAD 场景。
```

# 五、整体实验顺序

建议严格按这个顺序：

```text
Step 0：确认 71.950 可复现
Step 1：实验 1，只加 align loss，不改融合

Step 3：实验 4，加 top-k text gated fusion

```

不要一开始把 align loss、fusion、top-k 全部混在一起，否则结果不好时无法判断原因。
