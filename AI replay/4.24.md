我给你一个**明确的结论**，不再发散：

# 接下去就做这条主线

## **主线 C：残差式 classifier + 时序一致性**

并且把你之前提到的 **ranking / pairwise** 放进去当**第二个可加的损失**。
至于“特征投影对齐”，**先不作为主线做**。

---

# 为什么这样定

## 1. 为什么不把“特征投影对齐”放主线

因为我们前面的实验已经非常明确了：

* 只要测试时更新碰到 **feature/head/decoder/work-level feature**
* 性能就会**大幅崩掉**

所以如果你现在做“特征投影对齐”，无论名字听起来多好，本质上它都很容易又回到：

> **在测试时动表征层**

这跟我们已经证伪的方向是很接近的。

### 所以这个想法不是不能做

而是要**换一种落点**：

* 不做 **video feature projection alignment**
* 而做 **classifier / prototype space 的 alignment**

也就是说，把“投影对齐”从**特征层**，改成**分类器层 / 文本原型层**。
这样才安全。

---

## 2. 为什么“时序一致性更新”值得做

这个方向我也觉得很好，而且**非常贴 OAD**。

你现在最大的一个问题是：

* 当前 TTA 是**单窗口独立更新**
* 但 OAD 本质上是流式任务，前后窗口不该互相矛盾得太厉害

所以给 classifier-only 加一个**时序一致性约束**，是非常自然的。

而且它有两个优点：

* 不碰表征层
* 是 OAD 特有的，不只是调参

---

## 3. 为什么“残差式 classifier”是最合适的骨架

这点我现在最推荐。

因为当前 classifier-only 的问题是：

* 直接改原 classifier，虽然能涨到 **71.911**
* 但本质上风险还是大：伪监督一偏，原 classifier 就被拉走

所以更合理的做法是：

> **固定原 classifier 不动，只额外学一个很小的 residual classifier**

形式上就是：

[
\text{logits} = \text{logits}*{base} + \alpha \cdot \text{logits}*{res}
]

其中：

* `base classifier` 固定
* 测试时只更新 `residual classifier`
* `residual classifier` 初始化为 0，等价于“一开始完全不改原模型”

这个非常适合你现在的局面，因为它同时满足：

* 不碰 feature
* 比直接改 classifier 更稳
* 可以自然地接 ranking / temporal consistency / anchor 正则

---

# 所以我建议的最终路线是

## **C组主线：Residual Classifier TTA**

这是你后面真正的方法主线。

### C1：Residual classifier + 原来的 KL / text target

这是最基础版。

只改：

* 新增一个 residual classifier
* 原 classifier 固定
* 测试时只更新 residual classifier

目的：

* 验证 residual 骨架是否比“直接改 classifier”更稳

---

### C2：C1 + 时序一致性损失

在 C1 上再加一个 temporal consistency loss。

例如：

* 当前窗口更新后的预测，不要和前一窗口的 EMA 预测差太大
* 或者相邻窗口 top-k 分布尽量平滑

目的：

* 把“单窗口更新”升级成“流式更新”

这是我认为**最值得做的增强项**。

---

### C3：C1 + ranking / pairwise 损失

这个你之前提的我觉得可以做，但**要落在 logits / residual classifier 上**，不要落在 feature 上。

### 怎么做更合理

不是做 feature projection alignment，而是做：

* 根据文本 target distribution 选出 pseudo-positive 类
* 根据当前 logits 选 hard-negative 类
* 对它们做 pairwise ranking loss

形式类似：

[
\mathcal{L}*{rank} = \max(0, m - s*{pos} + s_{neg})
]

其中：

* `pos`：文本分布里高权重类别
* `neg`：当前预测里竞争最强的负类

目的：

* 不只是让分布整体接近文本 target
* 还显式拉开“文本支持类”和“混淆负类”的间隔

这个比单纯 KL 更像分类器优化。

---

### C4：C1 + temporal consistency + ranking

如果 C2 和 C3 单独都有效，再组合。

注意：

* 不要一开始就直接三者全上
* 要先做逐步消融，不然你没法判断到底是谁有效

---

# 那“特征投影对齐”怎么办

我的建议是：

## **现在不做 feature projection alignment**

但可以把它改造成：

### **classifier-prototype alignment**

不是对齐视频特征，而是对齐：

* residual classifier 的类别权重
* 文本类别原型（或投影后的文本原型）

比如：

[
\mathcal{L}_{proto} = 1 - \cos(W_c^{base} + \Delta W_c,; P(t_c))
]

其中：

* (P(t_c)) 是文本类别原型经过一个固定投影后的向量
* 对齐的是**分类器权重空间**
* 不是视频 feature

这样这个思路就还在，但不会踩到你已经证伪的 feature-level 更新。

不过这条我建议**排在 C2 / C3 后面**，不是第一优先级。
