有可行帮助，但要**抽象成轻量模块**，不能照搬 ALBEF 的完整大规模图文预训练。对你后续 OAD / LSTR 的实验来说，ALBEF 真正有价值的是这四句话：

> **不要直接 concat RGB/video 和 text 特征；先做 video-text 对齐，再做融合。**
> **融合时不要让所有类别文本都无脑参与，可以先筛 top-k，再细粒度交互。**
> **动作类别之间有天然 hard negatives，要专门利用相似动作文本做区分。**
> **文本描述可能不完整或有噪声，所以文本监督最好不要太硬。**

ALBEF 的依据是：作者明确指出图像特征和文本 token 原本处在不同空间，直接让多模态编码器学交互很困难，因此先用 ITC 对齐，再用 cross-attention 融合；ITC 还能帮助后续 ITM 找更有信息量的 hard negatives。

# 一、最值得你做的实验 1：Video-Text Contrastive Alignment

这是最可行、风险最低的实验。

## 核心想法

在你现有 LSTR/OAD 模型里，不要一开始就改结构。先加一个辅助损失，让视频/RGB 时序特征和动作类别文本特征对齐。

对应 ALBEF 里的思想是：

> ALBEF 先在单模态 image encoder 和 text encoder 的输出上做 ITC，让图文特征进入共同语义空间，再进行融合。

迁移到你这里就是：

[
F_v \rightarrow z_v
]

[
E_c \rightarrow z_c
]

然后让当前帧/窗口的视频特征 (z_v) 和 GT 动作类别文本 (z_c) 更接近，和其他类别文本更远。

## 具体怎么做

你已有：

* 视频/RGB 特征或 LSTR hidden feature：(H \in \mathbb{R}^{B \times T \times D})
* 类别文本特征：(E \in \mathbb{R}^{C \times D_t})

加两个小投影头：

[
z_v = \text{Proj}_v(H)
]

[
z_t = \text{Proj}_t(E)
]

然后做 normalized dot product：

[
s_{t,c}=z_v(t)^T z_t(c)
]

训练时对有动作标签的帧做 contrastive / CE：

[
L_{align}=CE(s_{t,:}, y_t)
]

如果一帧多标签，就改成 multi-positive contrastive 或 BCE 版本。

## 为什么这个最可行

它不需要重训大模型，不需要 BERT，不需要 multimodal encoder，也不改变主干结构。只是在原有分类 loss 旁边加一个轻量辅助 loss。

建议第一版设置：

[
L = L_{cls} + \lambda L_{align}
]

[
\lambda = 0.01, 0.03, 0.05
]

先小权重，不要让文本监督破坏原本 LSTR 分类能力。

## 你能怎么证明有效

除了最终 mAP，还可以记录：

1. GT 类别文本在 video-to-text similarity 里的 top-k hit rate；
2. 相似动作类别的混淆是否下降；
3. action-only frames 上 top-1/top-3 text retrieval 是否提升。

这个和你之前做 top-k hit rate 的思路很一致，落地难度低。

# 二、最值得你做的实验 2：Text-Guided Gated Fusion

这是更像“真正特征融合”的版本。

## 核心想法

ALBEF 不是只做对齐，它后面还有 multimodal encoder，用 cross-attention 让图像特征和文本特征融合。作者图 1 里明确展示了 image encoder、text encoder 和 multimodal encoder，文本特征在 multimodal encoder 每层通过 cross-attention 融合图像特征。

但你不适合直接搬 6 层 BERT multimodal encoder，因为太重，也不适合 OAD。

你可以做轻量版：

[
C_t = CrossAttn(Q=F_v, K=E_{topk}, V=E_{topk})
]

[
g = \sigma(MLP([F_v, C_t, F_v \odot C_t]))
]

[
F_{fuse}=LayerNorm(F_v+\lambda g C_t)
]

这里 (F_v) 是当前时间步/窗口的视觉特征，(E_{topk}) 是文本候选特征。

## 关键：一定要 gated residual

不要直接：

[
F_{fuse}=F_v+C_t
]

而是要：

[
F_{fuse}=F_v+\lambda g C_t
]

原因是你的视觉 baseline 已经比较强，文本一旦不准，直接加进去容易伤性能。gate 的作用是让模型自己判断文本是否该参与。

建议：

* gate 初始化接近 0；
* (\lambda) 初始设小，比如 0.1；
* 先只加一层 cross-attention；
* 只在 classifier 前融合，不动 LSTR 主体。

## 为什么这个实验值得做

因为它比“logits 后处理”更像老师想看的 **RGB/Text feature fusion**。
它不是最后把文本分数加到结果上，而是在特征层让文本参与视觉表示更新。

你可以这样命名：

> Text-Guided Gated Fusion，TGGF

或者：

> Align-before-Fuse Inspired Text-Guided Fusion

# 三、最值得你做的实验 3：Action Hard Negative Alignment

这个和 OAD 特别相关。

## 核心想法

ALBEF 的 hard negative 是：

> 语义相似但细节不匹配的负样本。模型用 ITC 相似度在 batch 内找 hard negatives，再用于 ITM 训练。

动作识别/OAD 里特别适合这个思想，因为动作类别本来就容易混：

* discus throw / hammer throw / javelin throw / shot put
* high jump / long jump
* diving / cliff diving
* basketball dunk / basketball shot
* cricket bowling / cricket shot

普通 CE loss 只告诉模型 GT 是对的，其他都是错的；但没有强调“最容易混的错类要重点区分”。

## 具体怎么做

先用类别文本特征算 class-class similarity：

[
S_{c,c'} = E_c^T E_{c'}
]

对每个类别 (c)，取 top-M 个相似但不是自己的类别作为 hard negatives。

训练时加 margin ranking loss：

[
L_{hn}=\max(0, m - sim(F_v,E_{pos}) + sim(F_v,E_{hard}))
]

意思是：

> 视频特征和正确动作文本的相似度，要比 hard negative 动作文本高至少一个 margin。

建议第一版：

* hard negative 数量：M=3 或 5；
* margin：0.1 或 0.2；
* loss 权重：0.01 或 0.03。

## 这个实验为什么可行

不需要改模型结构，只需要预先计算文本类别相似矩阵，然后训练时取相似负类即可。

这个实验很适合写成消融：

| 设置              | 作用     |
| --------------- | ------ |
| baseline        | 原始视觉模型 |
| + text align    | 文本语义对齐 |
| + hard negative | 区分相似动作 |
| + gated fusion  | 真正特征融合 |

# 四、可以做但放后面的实验：Top-k Text Candidate Fusion

这个是把 ALBEF 的 “ITC 粗筛 + ITM 精判” 转成 OAD 版本。

ALBEF 在检索任务里会先用 ITC 相似度筛 top-k，再用 ITM 做更细粒度排序，这样既有效又快。论文在下游检索策略里也说明，先计算 image-text similarity，再取 top-k candidates 计算 ITM 分数，可以避免对所有图文对都做重型融合。

迁移到你这里：

1. 当前视频/RGB 特征先和所有类别文本算相似度；
2. 选 top-k 类别文本；
3. 只对 top-k 文本做 cross-attention fusion；
4. 输出 refined logits。

这样比“所有类别文本都参与融合”更轻，也更符合 OAD 推理效率。

建议：

* top-k = 3 或 5；
* 先离线训练，不要一开始就做 TTA；
* 只在当前时间步/窗口上做，不能用未来帧。

这个方向和你之前 top-k calibration 很接近，但它更像 feature-level fusion，不只是 logits-level calibration。

# 五、MoD 的可行借鉴：不要完整照搬，做 MoD-lite

ALBEF 的 MoD 是为了解决 web 图文数据 noisy supervision：图文对弱相关，某些负文本其实也可能描述图像，MLM 中也可能有多个合理词，但 one-hot 标签会惩罚所有非 GT 预测。作者用 momentum model 生成 soft pseudo-targets 来缓解这个问题。

你的任务里也有类似问题：

* 类别文本太短；
* LLM evidence 可能有冗余；
* 一个动作有多个合理描述；
* 当前帧可能只包含动作的一部分；
* 某些相似动作不应该被过度强行推远。

但你不适合完整维护 ALBEF 那种 momentum multimodal model。你可以做轻量版：

## 方案 A：Semantic Label Smoothing

对 GT 类别给最大权重，同时给相似动作类别一点小概率。

比如 GT 是 discus throw：

* discus throw：0.90
* hammer throw：0.03
* javelin throw：0.03
* shot put：0.03
* others：接近 0

这样可以避免文本语义相近类被完全当成 0。

## 方案 B：EMA Teacher Soft Logits

维护一个 EMA classifier 或 EMA 模型输出，把它作为 soft target：

[
L = CE(y, p) + \alpha KL(p_{ema}||p)
]

这比完整 MoD 简单很多。

## 方案 C：Multi-text Prototype Averaging

每个动作类别不要只用一个类别名，而是多个文本描述：

* class name；
* prototype description；
* evidence description；
* LLM short definition。

然后平均或 attention 聚合成类别文本原型。

这个很适合你当前已经在做 class text bank 的情况。

# 六、不建议你做的部分

有些东西看起来高级，但对你现在不划算。

## 1. 不建议完整复现 ALBEF

ALBEF 是大规模 VLP 预训练，用 4M/14M 图文数据，模型包括 ViT、BERT text encoder、multimodal encoder 和 momentum model。它不是一个适合直接插进 LSTR 的小模块。

## 2. 不建议做 MLM

MLM 是预测 masked word，适合图文语言建模。你的 OAD 是动作检测，不是生成文本。你可以借鉴“文本语义一致性”，但不需要真的做 masked language modeling。

## 3. 不建议一开始做大 cross-attention encoder

6 层 multimodal encoder 对 OAD 太重，而且容易破坏已有 baseline。你现在更适合做：

* 一层 cross-attention；
* gated residual；
* top-k 文本候选；
* 小权重辅助 loss。

## 4. 不建议直接把所有类别文本拼进视觉特征

所有类别文本一起参与融合，计算量大，也容易引入无关文本噪声。应该先 top-k 或 gate。

# 七、我建议你后续实验路线

## 第一阶段：只验证文本对齐有没有用

最小改动：

[
L = L_{cls} + \lambda L_{align}
]

实验组：

| 实验                    | 内容                                 |
| --------------------- | ---------------------------------- |
| baseline              | 原始 LSTR/RGB 或双模态                   |
| + align loss          | 加 video-text contrastive alignment |
| + hard negative align | 加相似动作 hard negatives               |

观察：

* mAP 是否涨；
* text top-k hit 是否涨；
* 相似动作混淆是否下降。

这是最稳的一步。

---

## 第二阶段：做真正 feature fusion

在 classifier 前加：

[
F_{fuse}=LN(F_v+\lambda g CrossAttn(F_v,E_{topk}))
]

实验组：

| 实验                   | 内容                 |
| -------------------- | ------------------ |
| + concat             | 简单拼接 baseline      |
| + cross-attn         | 文本 cross-attention |
| + gated cross-attn   | 加 gate 和 residual  |
| + top-k gated fusion | 只融合 top-k 文本       |

如果 gated cross-attn 比 concat 稳，就可以说明你的融合不是简单堆特征，而是有选择地注入文本语义。

---

## 第三阶段：做 MoD-lite / soft text supervision

在已有 align/fusion 有效果后，再试：

| 实验                       | 内容           |
| ------------------------ | ------------ |
| semantic label smoothing | 给相似动作小概率     |
| EMA teacher KL           | 用 EMA 输出做软标签 |
| multi-text prototype     | 多描述文本原型      |

这一阶段主要解决文本不完整和相似动作过度惩罚问题。

---

# 八、你可以在组会里这样总结

> ALBEF 对我的实验启发主要不是复现完整 VLP 预训练，而是提供了一个可迁移的融合原则：视觉和文本特征在融合前需要先对齐，否则直接 concat 或 cross-attention 很难学习稳定的跨模态关系。因此，后续实验可以先在 OAD 模型中加入 video-text contrastive alignment，使视频时序特征和动作文本原型进入共同语义空间。其次，借鉴 ALBEF 的 hard negative 思想，利用动作文本相似度构造相似但错误的动作负类，增强模型对细粒度动作类别的区分能力。最后，在特征层面采用轻量 text-guided gated fusion，用 top-k 动作文本通过 cross-attention 影响视觉时序特征，并用 gate 控制文本注入强度，避免无关文本破坏原始视觉表示。MoD 部分不适合完整照搬，但可以借鉴为 semantic label smoothing、EMA teacher soft target 或 multi-text prototype，以缓解文本描述不完整和类别语义相近带来的硬标签问题。

我的最终建议是：**先做 “Alignment loss + Hard negative”，再做 “Gated fusion”，最后再考虑 soft supervision。**
这条路线最可行、改动最小，也最容易证明文本确实在特征层面帮助了 OAD。
