对，这组结果**不理想，但信息量很大**。它基本证明：**现在这个 adaptive 条件确实不够好，不是 adaptive 思路一定不行，而是“什么时候继续 step”这个判断规则没设计好。** 你这次跑的是 `max3 + rel_eps=0.05`，不同学习率下的结果都低于固定 `step=2, lr=4e-4` 的 71.950。日志里也能看到 6 组设置都已经正常跑完，并输出了 mAP 和 step 分布。

## 1. 这组结果说明了什么？

| adaptive 设置   |    mAP | avg steps | step 分布特点 |
| ------------- | -----: | --------: | --------- |
| max3, lr=4e-4 | 71.797 |    2.5767 | 大多数跑到 3 步 |
| max3, lr=1e-4 | 71.685 |    1.2749 | 大多数 1 步停  |
| max3, lr=8e-5 | 71.624 |    1.1389 | 基本 1 步停   |
| max3, lr=4e-5 | 71.557 |    1.0000 | 全部 1 步    |
| max3, lr=8e-6 | 71.527 |    1.0000 | 全部 1 步    |
| max3, lr=4e-6 | 71.523 |    1.0000 | 全部 1 步    |

最关键的不是 mAP 低，而是这个现象：

```text
lr 小的时候，adaptive 几乎全都停在 1 步；
lr 大的时候，adaptive 又大量跑到 3 步；
真正停在 2 步的窗口非常少。
```

但是我们前面已经验证过：

```text
固定 step=2 最好，step=1 不够，step=3 开始容易过拟合。
```

所以现在这个 adaptive 规则刚好避开了最优区域：
小学习率时变成“step=1”，大概率更新不足；大学习率时变成“step=3”，大概率更新过头。

这就是它没超过固定 step=2 的主要原因。

## 2. 为什么 adaptive 没有按设想变好？

理论上，adaptive 应该比固定 step 更灵活。但前提是：

> 判断“是否继续 step”的条件，要和真实 mAP 提升方向相关。

你现在的条件本质是看 KL loss 下降是否明显：

```text
如果 KL 还在下降，就继续；
如果下降不明显，就停。
```

问题在于，我们这里的 KL 目标是文本伪目标 Q，不是真实标签。继续降低 KL，不一定代表动作分类更对。之前 `max5, lr=4e-4` 掉到 70.825，就是很典型的例子：KL 继续优化，但 mAP 下降。

所以这个结果可以总结成一句：

> 当前 KL-loss-based adaptive 规则不能可靠判断真实预测是否变好；它更像是在判断“还能不能继续贴近文本伪目标 Q”，而不是判断“是否应该继续更新分类器”。

## 3. 下一步应该怎么改？

我觉得你说得对：**应该尝试多个 adaptive 条件**。但不要乱试，要围绕“为什么现在失败”来改。

### 方案 A：把 step=2 设为默认，而不是 step=1 起停

现在 `TTA_MIN_STEPS=1`，导致很多窗口 1 步就停。但我们已经知道 step=2 是最优锚点，所以 adaptive 应该改成：

```text
TTA_MIN_STEPS = 2
TTA_MAX_STEPS = 3
```

这样它回答的问题就变成：

```text
默认至少做 2 步；
只判断要不要额外做第 3 步。
```

这比现在合理很多。现在这个 adaptive 是在判断 1/2/3 步，但它实际上几乎只选了 1 或 3，反而绕开了最好的 2。

建议下一组不改代码，直接跑：

| min step | max step | rel eps |   lr |
| -------: | -------: | ------: | ---: |
|        2 |        3 |    0.01 | 4e-4 |
|        2 |        3 |    0.05 | 4e-4 |
|        2 |        3 |    0.10 | 4e-4 |
|        2 |        3 |    0.01 | 3e-4 |
|        2 |        3 |    0.05 | 3e-4 |
|        2 |        3 |    0.10 | 3e-4 |

这组最直接检验：**以 step=2 为基础，少量窗口加第 3 步是否有收益。**

### 方案 B：加入“可靠性条件”，不是只看 KL 下降

更合理的 adaptive 条件应该同时考虑：

```text
1. KL 是否还明显下降；
2. 视觉预测和文本 Q 是否一致；
3. 更新后预测是否变得过于尖锐或漂移；
4. top-1 类别是否稳定。
```

比如只有满足下面条件才允许第 3 步：

```text
KL 下降明显
并且更新后 top1 仍在原始 top-k 内
并且预测熵没有异常下降
并且参数漂移不太大
```

这需要稍微改代码，但方向比单纯 KL 早停更靠谱。

### 方案 C：step2 / step3 结果做回退或融合

还有一个更稳的做法：

```text
先做 step2；
如果尝试 step3 后 KL 下降但预测置信度异常变化，就回退到 step2；
或者直接平均 step2 和 step3 的 logits。
```

这个思路适合防止 step3 过拟合文本伪目标。

> 在 adaptive TTA 中，我们进一步测试了 `max_steps=3` 下不同学习率量级的表现。实验结果显示，低学习率下 adaptive 大量提前停止在 1 步，导致更新不足；高学习率下大量窗口继续到 3 步，导致过度贴合文本伪目标。最终所有设置均未超过固定 step=2 的最优结果 71.950。这说明当前基于 KL loss 下降的 adaptive step 条件不能可靠反映真实 mAP 的变化，后续需要将 step=2 作为默认更新强度，并引入预测稳定性或文本-视觉一致性作为继续更新的条件。
