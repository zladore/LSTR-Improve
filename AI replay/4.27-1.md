## 1. 我们当前任务的规划

目前结果可以总结成三条：

第一，**只做 additive logits 校准**的 A 组提升很小，双模态大概 +0.053 / +0.081，三模态大概 +0.060 / +0.093，说明“不更新参数”比较稳，但上限低。

> **只更新 classifier，不更新 backbone / decoder / adapter。**

所以后续主线应该是：

**固定方法主体不变：classifier-only + tta_kl_topk**

## 5. 你现在还需要重点搞清楚的 batch 细节

你老师问 batch 细节是很关键的，因为你的方法是**测试时按 batch 做一次或两次 classifier 更新**，所以 batch 怎么构造会直接影响结果。

你后面需要记录这些信息：

1. 一个 batch 里有多少 sample，通常可能是 `B=16`，但必须以代码为准。
2. 每个 sample 是不是一个滑动窗口，而不是一个完整视频。
3. 输入 shape：双模态一般是 RGB + Flow，三模态还多 Text。
4. target shape：很可能只对 work memory 的 32 帧计算预测，例如 `[B, 32, 22]`。
5. TTA 更新时，是用整个 `[B, 32, C]` 的 logits，还是只用最后一帧 logits。
6. classifier 更新完后，是**每个 batch 重置回原始权重**，还是一直累积更新到下一个 batch。这个非常重要。
7. `Q(s)` 是怎么构造的，top-k 取几个类，是否 detach，是否用了温度 τ。
8. 每一步更新前后，KL loss、entropy、top1 类别变化率、logits norm、grad norm 分别是多少。

后面你把代码发我，我建议直接给你加一个 `debug_tta_batch=True` 的开关，每隔若干 batch 打印一次这些信息。这样你不仅能给老师解释清楚，还能判断 71.911 是否还有继续提升空间。
